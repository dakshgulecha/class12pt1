Opening & Closing Files

The open() function creates a file object that links Python to a physical file on disk.

Syntax
file_obj = open("filename.txt", "mode")
file_obj.close()   # Always close to free memory
Best Practice — use with

The with statement automatically closes the file when the block ends, even if an exception occurs. Always prefer it over manual close().

Using the with clause
with open("data.txt", "r") as f:
    content = f.read()
# File is automatically closed here — no need for f.close()

The pickle Module

To store or retrieve Python objects (lists, dicts, tuples, etc.) from a binary file, you use pickle.

↓ Pickling (Serialization)
  • Converts a Python object → byte stream
  • Function: pickle.dump(data, file)
  • File must be opened in write binary mode: 'wb'
↑ Unpickling (Deserialization)
  • Converts byte stream → Python object
  • Function: pickle.load(file)
  • File must be opened in read binary mode: 'rb'
Writing a binary file
import pickle

student = {"name": "Aanya", "roll": 42, "grade": "A"}

with open("student.dat", "wb") as f:
    pickle.dump(student, f)    # Serialise and save
Reading a binary file (handling EOF)
import pickle

def DisplayAll():
    with open("student.dat", "rb") as f:
        try:
            while True:
                R = pickle.load(f)   # Reads ONE object at a time
                print(R)
        except EOFError:            # Raised when file ends
            pass
Must Know — EOFError

Binary files with multiple pickled objects require a try…except EOFError loop. When pickle.load() reaches the end of the file, it raises EOFError — without catching it, your program will crash.

CRUD Operations on Binary Files
Search

Iterate through all records and compare the target field against each record.

Search by roll number
def Search(target):
    found = False
    with open("student.dat", "rb") as f:
        try:
            while True:
                R = pickle.load(f)
                if R["roll"] == target:
                    print("Found:", R)
                    found = True
        except EOFError:
            pass
    if not found:
        print("Record not found")
Update  Delete

Since you can't edit a binary file in-place, you use the Temporary File Method:

1
Open original in rb and a new temp.dat in wb
2
Loop through all records using pickle.load()
3
Update: modify the matching record, then pickle.dump() to temp  |  Delete: simply skip the target record
4
Close both files
5
Use os.remove("original.dat") then os.rename("temp.dat", "original.dat")
Delete template
import pickle, os

def Delete(target_roll):
    records = []
    # Step 1: Load all records
    with open("student.dat", "rb") as f:
        try:
            while True:
                records.append(pickle.load(f))
        except EOFError:
            pass

    # Step 2: Write back — skipping the target
    with open("temp.dat", "wb") as f:
        for R in records:
            if R["roll"] != target_roll:
                pickle.dump(R, f)

    # Step 3: Replace original
    os.remove("student.dat")
    os.rename("temp.dat", "student.dat")

Opening CSV Files — The Crucial Nuance
Critical — newline='' when Writing

When opening a CSV file for writing, always pass newline=''. Without it, Python inserts an extra blank line between every row because of how Windows handles line endings.

Writing
F = open("data.csv", 'w', newline='')
Reading
F = open("data.csv", 'r')
Writing to a CSV File

Create a writer object first, then use it to write rows.

FunctionInputWritesNuance
csv.writer(file, delimiter)File objectWriter objectDefault delimiter is comma ','
writerow(list)A single ListOne rowEach list element becomes one field
writerows(nested_list)List of listsMultiple rows at onceBest for bulk/batch writing
Write example
import csv

data = [
    ["Name", "Roll", "Grade"],
    ["Aanya", 1, "A"],
    ["Rohan", 2, "B+"],
]

with open("students.csv", 'w', newline='') as f:
    writer = csv.writer(f)
    writer.writerow(data[0])       # Write header row
    writer.writerows(data[1:])     # Write all data rows at once
Reading from a CSV File

Create a reader object, then iterate over it with a for loop. Each row is returned as a List of strings.

Data Type Warning

CSV always returns data as strings. If you need to do arithmetic, convert first: int(row[1]), float(row[2]). Forgetting this is a very common exam mistake.

Read example with calculation
import csv

with open("marks.csv", 'r') as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)                          # Each row is a List
        total = int(row[1]) + int(row[2])  # Must convert to int
        print("Total:", total)

02 User-Defined Functions (UDF)
Defining a Function

Use the def keyword followed by the function name and parentheses (). The function body is indented below the def line.

Syntax
def function_name(parameters):
    # function body
    return result
The return Statement
Key Rule

If a return statement is not specified, the function returns None by default.

A function can return different types based on conditions.

Example — multiple return types
def factorial(n):
    if n < 0:
        return "Invalid argument"   # Returns a string
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result                   # Returns an integer
03 Arguments & Parameters
Remember the Distinction

Parameters → variables listed in the function definition
Arguments → values passed during the function call

Positional Arguments

Must be passed in the correct order/position. Swapping positions gives logically incorrect output even if the code runs.

def minus(a, b):
    return a - b

minus(10, 20)   # Returns -10  ✅ (intended)
minus(20, 10)   # Returns  10  ❌ (wrong order — wrong result)
Keyword (Named) Arguments

Parameter names are used explicitly during the call. Order can be changed — Python matches by name, not position.

def student(firstname, grade):
    print(firstname, grade)

student(grade='Seventh', firstname='John')   # ✅ Order changed — still works
Default Parameters

Values assigned during the function definition using =. Default arguments are optional during the call — if provided, they override the default.

def greet(name, msg="Hello"):
    print(msg, name)

greet("Alice")           # Uses default → "Hello Alice"
greet("Alice", "Hi")     # Overrides default → "Hi Alice"
Critical Order Rule

Non-keyword arguments cannot follow keyword arguments in a function call.

student(firstname='John', 'Seventh')   # ❌ SyntaxError
student('John', grade='Seventh')       # ✅ Correct
Arbitrary Arguments (Variable-Length)
SyntaxType StoredDescription
*argsTupleAccepts any number of non-keyword arguments
**kwargsDictionaryAccepts any number of keyword arguments
 
def show_args(*args, **kwargs):
    print(args)    # ('a', 'b', 'c')  → Tuple
    print(kwargs)  # {'x': 1, 'y': 2} → Dictionary

The global Keyword

To modify a global variable from inside a function, you must declare it with the global keyword.

G = 10

def modify():
    global G       # Declare intent to modify global G
    G = G + 10     # ✅ Works fine

modify()
print(G)           # 20
UnboundLocalError — A Common Exam Trap
Exam Trap

This error occurs when you try to read a variable inside a function that also has an assignment to the same variable name later in the function — without declaring global.

Python sees the assignment G = G + 10 and treats G as a local variable for the entire function — even the print(G) line before the assignment.

❌ Causes UnboundLocalError
G = 10

def myfun():
    print(G)    # ❌ Error!
    G = G + 10  # treats G as local
✅ Fix with global
G = 10

def myfun():
    global G    # Declare first
    print(G)
    G = G + 10

01 Core Concepts
What is an Exception?

Exception handling is a mechanism to manage errors that occur during program execution without crashing it.

An exception is an error that occurs during run time that cannot be handled using the normal flow of control.

Critical Distinction

Compile-time errors or syntax errors (like missing colons or incorrect indentation) are NOT exceptions. Exception handling does not handle syntax errors.

Common Exceptions
ExceptionCauseExample
ZeroDivisionErrorDividing a number by zeroc = 5 / 0
ValueErrorPassing a value of the wrong type or formatint("hello")
NameErrorUsing a variable that hasn't been definedprint(x) — if x was never assigned
02 The Exception Handling Blocks

Python uses four blocks to test, catch, and resolve errors.

try Block

Purpose: Lets you test a block of code for errors. Python executes the try block first. If an error occurs, it immediately stops and switches to the matching except block.

try:
    y = int(input("Enter a number: "))
    # If this fails, the lines below won't run
    print(y * 2)
except Block

Purpose: Handles the error. You can have multiple except blocks to handle specific error types. Place specific exceptions before a generic except: block. A generic except: acts as a catch-all for any error not previously specified.

Order Rule

Always put specific except blocks (like NameError) before the generic except:. Python checks them top-to-bottom and stops at the first match.

try:
    y = int(input("Enter a number: "))
    print(x)
except NameError:
    print("Variable x is not defined")   # Specific handler first
except:
    print("Something else went wrong")   # Generic catch-all last
else Block

Purpose: Executes only when there is no error in the try block. Runs after the try block finishes successfully, but before the finally block.

try:
    result = 10 / 2
except ZeroDivisionError:
    print("Cannot divide by zero")
else:
    print("Success! Result is", result)   # Only runs if no exception
finally Block

Purpose: Executes regardless of whether an exception occurred or was handled. This block is always the last to run. Typically used for clean-up tasks like closing files or printing goodbye messages.

try:
    f = open("data.txt")
except FileNotFoundError:
    print("File not found")
finally:
    print("Good bye")   # Runs no matter what ✅
Block Summary
BlockWhen it runsTypical Use
tryAlways — firstCode that might raise an exception
exceptOnly if an exception occursHandle/report the error

finallyAlways — lastClean-up (close files, print goodbye)
03 Flow of Execution

The execution path depends on whether an exception is raised inside try.

Scenario A — No Exception
try ✔ (runs fully)
else ✔ (runs)
finally ✔ (always runs)

Path: try → else → finally

Scenario B — Exception Occurs
try ⚠ (stops at error)
matching except ✔
else ✘ SKIPPED
finally ✔ (always runs)

Path: try → except → finally

Key Point

The else block is always skipped when an exception occurs. finally runs in both scenarios, no exception or exception.

04 Comprehensive Example

This example combines all four blocks and multiple except handlers:

try:
    a = int(input("Enter a number: "))
    b = int(input("Enter a number: "))
    c = a / b               # Could raise ZeroDivisionError or ValueError

except ZeroDivisionError:
    print("Divide by zero error")        # Specifically catches division by 0

except ValueError:
    print("Input value is not correct")  # Specifically catches bad input type

except:
    print("Some other error occurred")   # Catch-all for anything else

else:
    print("No error - calculation successful")   # Only runs if no except ran

finally:
    print("Good bye")                    # Always runs, no matter what
